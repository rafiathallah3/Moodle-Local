# Reflective Critic Agent: The AI Quality Gate

The **Reflective Critic Agent** serves as the primary "Quality Control" layer of the AFS v2 pipeline. It ensures that any content generated by the AI (whether a grade or a lesson) meets pedagogical standards and security policies before it reaches the student.

## 📍 Where is it triggered?

The Critic is NOT called manually by PHP. Instead, it is triggered **automatically** within the Python Agentic Graph (`main.py`) whenever an analysis or generation step finishes.

### 1. After Grading (`agents/analyzer.py`)
At the end of the `AnalyzerAgent.process()` method:
```python
# File: agents/analyzer.py
state["scoring_result"] = scoring
state["next_node"] = "critic"  # <--- Triggers the Critic next
```

### 2. After Content Generation (`agents/lesson_generator.py`)
At the end of the `LessonGeneratorAgent.process()` method:
```python
# File: agents/lesson_generator.py
state["content_package"] = content
state["next_node"] = "critic"  # <--- Triggers the Critic next
```

### 3. The Execution Loop (`main.py`)
The `main.py` script contains the logic that sees the `next_node` instruction and routes the `AgenticState` to the `ReflectiveCriticAgent`.

---

## 🔍 Core Responsibilities

The logic inside `agents/reflective_critic.py` performs four distinct "Reflection" checks:

| Check | Description | Why it matters |
| :--- | :--- | :--- |
| **Leakage Detection** | Scans the AI's explanation for code blocks or phrases like "The answer is..." | Prevents the AI from accidentally giving the solution to a quiz. |
| **Scoring Confidence** | Verifies the `confidence` score from the Analyzer. | If the AI is "unsure" about a grade, the Critic flags it for revision. |
| **Socratic Review** | Checks for "active learning" markers (questioning vs lecturing). | Ensures the AI acts like a tutor, not just a textbook. |
| **CFF Policy Check** | Ensures compliance with "Cognitive Forcing Functions." | Keeps the AI locked behind rules if the student hasn't met certain criteria. |

---

## 🔄 The Self-Correction Loop

One of the most powerful features of the Critic is its ability to **reject** a response.

1. **Failure detected**: If a leakage or pedagogical issue is found, the Critic sets `revision_needed = True`.
2. **Loop Back**: The graph routes the state BACK to the Analyzer or Lesson Generator.
3. **Instructional Hint**: The Critic provides a specific "hint" to the next agent (e.g., *"Make it more concise and add a Socratic question"*).
4. **Maximum Iterations**: The system allows up to **2 revisions** to ensure efficiency while maintaining high quality.

---

## 📈 Impact on Moodle
All Critic decisions—including if it caught a "leak" or forced a "revision"—are logged in the `mdl_local_orchestrator_log` table under the `policy` and `agents_called` columns. This gives teachers a full audit trail of the AI's internal quality control process.
